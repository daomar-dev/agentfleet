AgentFleet

For developers running agents on more than one machine

Run one prompt across every device you own.

AgentFleet turns laptops, desktops, and lab machines into a parallel coding-agent fabric using OneDrive as the coordination layer — no server, queue, SSH mesh, or control plane.

npx -y @daomar/agentfleet run

Start this on each machine, then submit once from CLI or browser.

为拥有多台机器并运行 AI Agent 的技术人员设计

一次提交，让所有设备并行处理。

AgentFleet 使用 OneDrive 作为协调层，把笔记本、台式机、实验室机器变成并行编码智能体网络——无需服务器、队列、SSH 组网或控制平面。

npx -y @daomar/agentfleet run

在每台机器启动它，然后通过 CLI 或浏览器提交一次任务。

No control plane

No backend, scheduler, database, queue, or inbound port. If OneDrive sync works, the fleet can coordinate.

Parallel agent output

Every node runs its own local agent command and writes hostname-scoped results, so you can compare outputs side by side.

Firewall-friendly

Tasks and results stay in your Microsoft 365 / OneDrive environment. The web app calls Microsoft Graph directly.

How it works

Submit a task once. Each running machine sees the immutable task file, executes locally, and writes back a separate result file.

Machine A          Machine B          Machine C
agentfleet run     agentfleet run     agentfleet submit
     │                  │                   │
     └──────────┬───────┴──────────┬────────┘
                ▼                  ▼
        OneDrive / AgentFleet shared workspace
           tasks/  → immutable task JSON
           output/ → per-machine results

无控制平面

没有后端、调度器、数据库、队列或入站端口。只要 OneDrive 能同步，集群就能协调。

并行 Agent 输出

每个节点运行自己的本地 agent 命令，并写入带主机名前缀的结果，方便横向比较。

适合防火墙内环境

任务和结果留在你的 Microsoft 365 / OneDrive 环境中。Web 应用直接调用 Microsoft Graph。

工作原理

提交一次任务。每台运行中的机器读取不可变任务文件，在本机执行，并写回独立结果文件。

机器 A             机器 B             机器 C
agentfleet run     agentfleet run     agentfleet submit
     │                  │                   │
     └──────────┬───────┴──────────┬────────┘
                ▼                  ▼
        OneDrive / AgentFleet 共享工作区
           tasks/  → 不可变任务 JSON
           output/ → 每台机器独立结果
