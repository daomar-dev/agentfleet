Lattix

Distributed Agent Orchestration Without a Control Plane

A free, open-source, decentralized multi-machine coding agent fabric built on OneDrive sync.

Overview

Lattix is a decentralized, multi-machine coding agent orchestration tool. It lets you fan out coding tasks across all of your machines simultaneously. Submit a task from any machine, and every machine running Lattix will see the same task, execute it locally with its installed coding agent, and write back machine-scoped results.

Unlike traditional orchestration platforms that require a central scheduler, message broker, or cloud-hosted control plane, Lattix uses OneDrive file sync as the sole coordination mechanism. There are no servers to run, no databases to maintain, and no additional infrastructure to deploy.

How It Works

  1. Your machines share a synced OneDrive workspace.
  2. Lattix exposes stable local paths under ~/.lattix/, with tasks/ and output/ symlinked to your OneDrive.
  3. Any machine can create a task in the shared tasks/ directory — via CLI or the web dashboard.
  4. Every machine running lattix run picks up that task independently and executes it with its locally installed coding agent.
  5. Results land in a shared output/ directory with hostname-prefixed files to avoid collisions.

This decentralized architecture means no single point of failure. If one machine is offline, the others still execute the task. There is no leader election, no quorum, and no consensus protocol — just file-based coordination through a battle-tested sync engine.

Architecture

~/.lattix/
├── config.json          # Local machine config (not synced)
├── processed.json       # IDs of tasks already executed
├── tasks/ → OneDrive    # Symlink to <OneDrive>\Lattix\tasks
│   ├── task-001.json
│   └── task-002.json
└── output/ → OneDrive   # Symlink to <OneDrive>\Lattix\output
    ├── task-001/
    │   ├── DESKTOP-A-result.json
    │   └── LAPTOP-B-result.json
    └── task-002/
        └── ...

The CLI is built with Node.js and TypeScript. It uses a polling-based task watcher that checks for new task files at a configurable interval. Task files are immutable JSON documents — once written, they are never modified. Only tasks that arrive after the daemon starts are processed, ensuring safe restarts and new machine onboarding.

Security Model

Lattix is designed with a zero-infrastructure security model:

  • No tunnels — machines never open inbound connections. There is nothing to attack from the outside.
  • No exposed ports — no listening services, no open ports, no attack surface.
  • No data movement — data never leaves the Microsoft 365 tenant boundary.
  • No central server — coordination happens through OneDrive sync, already approved by your organization's IT policies.

The web dashboard authenticates via Microsoft Entra ID using the PKCE authorization code flow. MSAL.js manages tokens in the browser with delegated permissions (Files.Read, Files.ReadWrite, User.Read). All API calls go directly to Microsoft Graph — no Lattix backend is involved.

Getting Started

Install and run Lattix on your first machine:

npx -y lattix run

This auto-detects OneDrive, creates the workspace, and starts watching for tasks. Repeat on every machine you want to enroll.

Submit a task from any machine:

lattix submit --prompt "Add error handling to all API endpoints" --title "Error handling"

Or use the web dashboard to submit tasks and monitor results from any browser.
